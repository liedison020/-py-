###一些感悟与思考###
不是因为你够格了才去做项目，而是在做项目的过程中变得够格的
2026/4/14
要多多利用ai，拥抱ai时代
  今天跟一个大四保研到长安大学的学姐讨论了有关于肌电信号来控制机械的一个本科毕业设计项目，发现她在数据以及结果呈现的ui界面还是有些不尽如人意。
我推荐她用codex等大模型来为她设计一下ui，因为我最近看到观察到的现象发现ai对于互联网企业的前端开发是有毁灭性的打击的，你把你的需求丢给ai，它所设计出来的ai界面真的很简洁美观，有国外设计美学的味道，经过几轮的debug；需求添加，他所给你生成的网站可以满足很多的需求，是给软件工程师减负了，同时也削减了他们的工作岗位，对于经验的学习，人是比不过ai的。模型的迭代太快了，以至于程序员还没来得及开始进步就被代替，可能是企业为了降本增效。毕竟只需花费一些算力成本就可以完成80%的功能，留下来的不是码农，而是真正核心能提出来改进方法的人才。ai啊请等等你的人民？
工程师的作用就是优化，实现那些理想中的事情，为大众所用。工程的本质就是一个发现问题，解决问题，实现问题的步骤。知道学姐的机电控制项目之后，我有了一下几点思考
1.关于输入端的信号与处理，中间还经过了一个解析的步骤，耗时很长。基本上方案的实现是：收到信号——处理信号（拟合所给的方案）；收集信号——再次处理信号（分辨是握拳还是松手）——实现电机的起关。第一这是两个步骤，感觉有点a-b，b-c，是否可以让他们连成一条线呢？我观察到信号的处理时间有点慢，我是不是可以通过软件方面的算法来优化信号处理方法，她的代码一定不是最优解，py还是可以进行改进
2.关于电机，我认为如果要在她的基础上进行改进的话，一定是要控制一个复杂电机的，我认为单纯的01信号不足以满足实际需求，但我也要对成果的不可见性有心理准备
3.关于测量本身方面，肌电信号是一个实体，我们能不能把目光放到更远，试着讨论脑电信号的处理，来讨论关于脑电的信号来操作机械臂一类的东西。
如果要讨论脑控制的话，就会有不得不面对的问题，但我也要明白我自己所具有的优势
优势：我可以询问哥，看看他的想法，因为他毕竟是内行，一定比我所理解的更深，或许可以从他那里获得数据集什么的来可以减少工作量，因为他有着受试者的数据，可以减少我第一步添加数据集的前期与分析工作
问题：1.我怎么来优化算法让脑信号实现。到底要讨论哪种波
      2.信号系统的处理我不太明白，相当于现在还是根据别人的论文来实现，还是不懂得到底是什么信号来控制
      3.硬件是一个大问题，基于现在的条件，做一个所谓的肯定是不现实，况且这需要有医疗器械什么的，都是我需要面对的问题
      4.我要去Google里面找一找相关的论文，看看是不是有人做过。如果有人做过，我第一要尝试复现：第二我要思考我有什么可以创新的点，可以提出哪些改进，可以将什么东西运用到实际。是不是可以写一篇论文呢？

就讨论这么多吧 还是那句话：不是因为你够格了才去做项目，而是在做项目的过程中变得够格的！！！
